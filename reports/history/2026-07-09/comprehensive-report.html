潜在リスク Top 5

1. 公開境界の露出 Low

NSG の高リスクルールは 0 件と良好ですが、Public IP 件数が未取得のため、外部からのアクセスリスクは不明です。公開管理ポート（22, 3389）へのアクセス制御は継続的に監視してください。

推奨: Azure Bastion と Just-In-Time (JIT) VM アクセス、Application Gateway WAF の導入。

  • Azure Bastion の概要
  • Just-In-Time VM アクセス
  • Application Gateway WAF

2. 特権ロールの恒久付与と blast radius High

Owner 割り当てが 13 件、孤児候補が 40 件と多く、特権アクセスの恒久付与によるリスクが高い状態です。アクセスレビューや PIM の導入で blast radius を縮小すべきです。

推奨: Azure AD Privileged Identity Management (PIM)、マネージド ID の活用、定期的なアクセスレビュー。

  • Azure AD Privileged Identity Management
  • マネージド ID の概要
  • アクセスレビューの概要

3. Defender for Cloud のセキュリティ態勢ギャップ High

Defender for Cloud の Unhealthy 件数は 80 件で、そのうち High Severity が 32 件存在します。ベースラインの不備や未対応の脆弱性が多数あります。

推奨: Foundational CSPM の導入、Secure Score の月次レビューと改善活動。

  • Defender for Cloud の開始方法
  • Secure Score の活用
  • クラウドセキュリティ態勢管理 (CSPM)

4. Advisor フィードバックループ不在 High

Azure Advisor の High Impact 推奨が 25 件存在し、特に「Trusted Launch foundational excellence」の未対応が目立ちます。Advisor の推奨を活用した定期的な運用改善が必要です。

推奨: Advisor 推奨の月次レビュー、Cost Management との連携。

  • Azure Advisor の概要
  • Cost Management の活用
  • 運用の優秀性フレームワーク

5. ガバナンス基盤の弱さ Medium

タグ付与率は 50.0% と低く、Storage の冗長性設定や Public IP

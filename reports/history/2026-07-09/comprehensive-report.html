Azure 総合運用監査レポート

生成日時: 2026-07-09 04:17:50 UTC

エグゼクティブサマリ

本レポートは、現在ログイン中の Azure サブスクリプションにおけるリソース、RBAC、NSG、Defender for Cloud、Azure Advisor の横断的な棚卸しとリスク分析を実施した結果をまとめています。

主要数値サマリ

  • リソース総数: 51 件 / リソースグループ数: 7 / リージョン数: 5
  • VM 件数: データなし / Storage (LRS) 件数: データなし / Public IP 件数: データなし
  • タグ付与率: 45.1%
  • RBAC 総割り当て数: 未取得 / Owner 割り当て数: 13 / UAA (User Assigned Admin) 割り当て数: 2 / サービスプリンシパル割り当て数: 未取得 / 孤児候補件数: 40 / マネージドグループスコープの Owner 数: 未取得
  • NSG 数: 未取得 / 総ルール数: 未取得 / 高リスクルール件数: 0
  • Defender for Cloud Unhealthy 件数: 79（High: 31、Medium: 未集計、Low: 未集計）
  • Defender for Cloud 最頻出推奨事項: Update libpython3.10-stdlib_for_linux 等のセキュリティ更新
  • Azure Advisor 件数: 未集計（5カテゴリ内訳: データなし）、High Impact 件数: 25
  • Azure Advisor 最頻出問題: Linux 仮想マシンに Azure Disk Encryption または EncryptionAtHost の有効化推奨
  • Defender for Cloud および Azure Advisor は有効かつデータ取得可能

主要改善領域

  1. 公開境界の強化 - 公開IPやNSGの高リスクルールは0件ですが、Public IP 件数未取得のため注意が必要です。
  2. 特権境界の管理 - Owner割り当て13件、孤児候補40件と特権ロールの恒久付与リスクが存在します。
  3. 監視フィードバックの活用 - Defender for CloudのHighリスク31件、AdvisorのHigh Impact 25件を踏まえた継続的なレビューが必要です。
  4. ガバナンス基盤の強化 - タグ付与率45.1%と低めであり、タグ必須化や冗長性設定の見直しが望まれます。

潜在リスク Top 5

1. 公開境界の露出 Low
NSGの高リスクルールは 0件 であるものの、Public IP 件数は未取得のため、公開境界の実態把握にギャップがあります。公開IPの存在は潜在的な攻撃対象となるため、BastionやJITアクセス、Application Gateway WAFの導入を推奨します。
Azure Bastion JIT VM Access Application Gateway WAF
2. 特権ロールの恒久付与と blast radius Medium
Owner 割り当てが 13件、孤児候補が 40件 存在し、特権ロールの恒久付与によるリスクが懸念されます。PIM（Privileged Identity Management）やマネージドIDの活用、定期的なアクセスレビューを実施し、権限の最小化を推進してください。
Azure AD PIM マネージドID アクセスレビュー
3. Defender for Cloud のセキュリティ態勢ギャップ High
Defender for Cloud の Unhealthy 件数は 79件、うち High Severity が 31件 あります。セキュリティベースラインが不十分であり、Foundational CSPM（Cloud Security Posture Management）導入と Secure Score の月次レビューを強く推奨します。
Secure Score Defender CSPM 優先推奨事項
4. Advisor フィードバックループ不在 High
Advisor の High Impact 件数は 25件 あり、Linux VM のディスク暗号化未設定など重要な問題が多数検出されています。Advisor の推奨事項を活用した月次レビュー体制と Cost Management 連動による運用改善を推奨します。
Azure Advisor Cost Management Advisor ベストプラクティス
5. ガバナンス基盤の弱さ Medium
タグ付与率は 45.1% と低く、ストレージの冗長性設定や Public IP 管理も不十分な可能性があります。Azure Policy によるタグ必須化、タグ継承設定、ZRS/GZRS などの冗長性向上施策を推奨します。
Azure Policy

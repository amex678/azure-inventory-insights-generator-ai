1. 公開境界の露出 Low
本サブスクリプションでは高リスク NSG ルールは 0 件、Public IP 件数は未取得ですが、公開境界の管理は常に重要です。推奨される対策は Bastion と JIT (Just-In-Time) アクセスの導入、Application Gateway の WAF 利用です。

2. 特権ロールの恒久付与と blast radius Medium
Owner 割り当て数は 13 件、UAA 割り当て数は 2 件、孤児候補は 40 件と多く、特権ロールの恒久付与によるリスクが懸念されます。推奨は PIM (Privileged Identity Management)、マネージド ID の活用、Access Review の定期実施です。
Azure AD PIM マネージド ID Access Reviews
3. Defender for Cloud のセキュリティ態勢ギャップ High
Defender for Cloud の Unhealthy 件数は 155 件、うち High Severity は 39 件にのぼります。セキュリティ態勢に明確なギャップが存在し、Foundational CSPM の導入と Secure Score の月次レビューを強く推奨します。
Defender for Cloud 入門 Secure Score CSPM (Cloud Security Posture Management)
4. Advisor フィードバックループ不在 High
Advisor の High Impact 件数は 29 件あり、重要な運用改善提案が多数存在します。月次レビューの実施と Cost Management 連動による継続的改善を推奨します。
Azure Advisor 概要 Cost Management 連携
5. ガバナンス基盤の弱さ Medium
タグ付与率は 52.0% と半数程度であり、ストレージ冗長性や Public IP 管理も未最適の可能性があります。Azure Policy によるタグ必須化、Tag Inheritance の活用、ZRS/GZRS への冗長性移行を推奨します。
Azure Policy タグ付与のベストプラクティス

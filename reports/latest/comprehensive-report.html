  • リソース総数: 75 件 / リソースグループ数: 9 / リージョン数: 5
  • VM 件数: データなし / Storage (LRS) 件数: データなし / Public IP 件数: データなし
  • タグ付与率: 52.0%
  • RBAC 総割り当て: データなし / Owner 割り当て: 13 / UAA 割り当て: 2 / 孤児候補件数: 40 / マネージドグループスコープの Owner: データなし
  • NSG 数: データなし / 総ルール数: データなし / 高リスクルール件数: 0
  • Defender for Cloud Unhealthy 件数: 178（High: 53、Medium: 未集計、Low: 未集計）
  • Defender 最頻出推奨事項: Update linux-image-generic_for_linux 等の脆弱性更新
  • Advisor 件数: データなし（5カテゴリ内訳: データなし）、High Impact 件数: 31
  • Advisor 最頻出問題: Linux VM に対する EDR ソリューション未導入、ストレージアカウントのゾーン冗長化未設定
  • Defender for Cloud は有効でありデータ取得済み。Az.Advisor は有効でデータ取得済み。

主要改善領域

  1. 公開境界：NSG 高リスクルールは 0 件ですが、Public IP 件数の詳細が未取得のため、公開境界の露出リスクは要監視。
  2. 特権境界：Owner 割り当て 13 件、孤児候補 40 件と特権ロールの恒久付与と blast radius の拡大が懸念されます。
  3. 監視フィードバック：Defender for Cloud で 178 件の Unhealthy 状態（High 53 件含む）、Advisor で High Impact 31 件と監視フィードバックの活用強化が必要です。
  4. ガバナンス基盤：タグ付与率 52.0% と半数程度に留まっており、タグ管理強化やストレージ冗長性の見直しが望まれます。

潜在リスク Top 5

1. 公開境界の露出 Low

NSG 高リスクルール件数は 0 件であるものの、Public IP 件数の詳細が未取得のため、公開境界の露出リスクは完全には把握できていません。公開管理ポート（22, 3389）へのアクセス制御強化が推奨されます。

推奨: Azure Bastion と Just-In-Time (JIT) VM アクセス、Application Gateway の Web Application Firewall (WAF) の導入。

参考リンク： Azure Bastion の概要 / JIT VM アクセス / Application Gateway WAF

2. 特権ロールの恒久付与と blast radius Medium

Owner 割り当てが 13 件、孤児候補が 40 件と多く、特権ロールの恒久的な付与や管理されていないアカウントの存在により、権限の濫用リスクが高まっています。

推奨: Privileged Identity Management (PIM) の導入、マネージド ID の活用、定期的な Access Review の実施。

参考リンク： Azure AD Privileged Identity Management / マネージド ID の概要 / Access Reviews の概要

3. Defender for Cloud のセキュリティ態勢ギャップ High

Defender for Cloud の Unhealthy 件数は 178 件、そのうち High Severity が 53 件含まれており、セキュリティ態勢に重大なギャップがあります。

推奨: Foundational CSPM の導入、Secure Score の月次レビューを実施し、優先度の高い推奨事項から対応を進めること。

参考リンク： Azure Defender for Cloud の概要 / Secure Score の活用 / 優先推奨事項の管理

4. Advisor フィードバックループ不在 Medium

Advisor の High Impact 件数は 31 件あり、重要な改善点が多数検出されていますが、月次レビューや Cost Management との連携が不足している可能性があります。

推奨: Advisor 推奨事項の月次レビューの実施、Cost Management との連携強化。

参考リンク： Azure Advisor の概要 / Cost Management の活用

5. ガバナンス基盤の弱さ Medium

タグ付与率は 52.0% と半数程度に留まっており、ストレージの冗長性設定や Public IP 管理も含めてガバナンス基盤の強化が必要です。

推奨: Azure Policy によるタグ必須化、Tag Inheritance の実装

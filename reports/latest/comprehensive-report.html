エグゼクティブサマリ

本レポートは、現在ログイン中の Azure サブスクリプションにおけるリソース、RBAC、NSG、Defender for Cloud、Azure Advisor の横断的な棚卸しとリスク分析を実施した結果をまとめたものです。

  • リソース総数: 51 件 / リソースグループ数: 7 / リージョン数: 5
  • VM 件数: 未取得（詳細データなし） / Storage (LRS) 件数: 未取得 / Public IP 件数: 未取得
  • タグ付与率: 45.1%
  • RBAC 総割り当て数: 未取得 / Owner 割り当て数: 13 / UAA (User Assigned Admin) 割り当て数: 2 / サービスプリンシパル割り当て数: 未取得 / 孤児候補割り当て数: 40 / マネージドグループスコープの Owner 数: 未取得
  • NSG 数: 未取得 / 総ルール数: 未取得 / 高リスクルール件数: 0
  • Defender for Cloud Unhealthy 件数: 79（High: 31、Medium: 未集計、Low: 未集計）
  • Defender for Cloud 最頻出推奨事項: Linux VM の複数パッケージ更新（High Severity 複数）
  • Azure Advisor 件数: 未集計（5カテゴリ内訳未取得） / High Impact 件数: 25
  • Azure Advisor 最頻出問題: Linux VM に対する Azure Disk Encryption または EncryptionAtHost 未設定
  • Defender for Cloud および Azure Advisor は有効かつ取得可能

主要改善領域

  1. 公開境界の強化：NSG 高リスクルールは 0 件ですが、Public IP 件数は未取得のため要確認。Bastion や JIT、Application Gateway WAF の導入推奨。
  2. 特権境界の管理強化：Owner 割り当て 13 件、孤児候補 40 件と多く、PIM や Access Review の適用が必要。
  3. 監視フィードバックの充実：Defender for Cloud で High Severity の未対応推奨が多数。Foundational CSPM と Secure Score の月次レビューを推奨。
  4. ガバナンス基盤の強化：タグ付与率 45.1% と低く、Azure Policy やタグ継承の導入、Storage 冗長性の見直しが必要。

潜在リスク Top 5

1. 公開境界の露出 Medium

NSG の高リスクルールは 0 件と安全ですが、Public IP 件数は未取得のため、外部からのアクセス経路の把握が不十分です。

推奨: Azure Bastion と JIT (Just-In-Time) VM アクセスの導入、Application Gateway の WAF 機能活用。

参考リンク: Azure Bastion / JIT VM アクセス / Application Gateway WAF

2. 特権ロールの恒久付与と blast radius High

Owner 割り当てが 13 件、孤児候補が 40 件と多く、不要な権限が放置されている可能性があります。

推奨: Azure AD Privileged Identity Management (PIM) の導入、マネージド ID の活用、定期的な Access Review の実施。

参考リンク: Azure AD PIM / マネージド ID / Access Review

3. Defender for Cloud のセキュリティ態勢ギャップ High

Defender for Cloud の Unhealthy 件数は 79 件で、そのうち High Severity が 31 件存在します。Linux VM の複数パッケージ更新が未対応です。

推奨: Foundational CSPM の導入、Secure Score の月次レビューを実施し、継続的なセキュリティ態勢の改善を図る。

参考リンク: Defender for Cloud 概要 / Secure Score / CSPM (Cloud Security Posture Management)

4. Advisor フィードバックループ不在 High

Azure Advisor の High Impact 件数は 25 件で、Linux VM の暗号化未設定が多く見られます。

推奨: Advisor 推奨事項の月次レビューを実施し、Cost Management と連動した改善活動を継続的に行うこと。

参考リンク: Azure Advisor / Cost Management / Advisor ベストプラクティス

5. ガバナンス基盤の弱さ Medium

タグ付与率が 45.1% と低く、Storage の冗長性設定や Public IP 管理の強化が必要です。

推奨: Azure Policy によるタグ必須化、タグ継承の導入、Storage 冗長性の ZRS/GZRS への移行検討。
